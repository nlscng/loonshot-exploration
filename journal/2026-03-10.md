### Vision Shift: Redefining "Meta"

Decided the v1 definition of "meta" (raw card popularity) is too shallow. V2 redefines meta as **card-in-context**: which decks/archetypes run a card, what synergizes with it, why it matters in the current format.

Researched data sources and wrote a scoping document: `explorations/2026-03-10-meta-runner-v2-scope.md`

**Data sources identified:**
- **AlwaysBeRunning.net** — ✅ public API, tournament results with standings + NRDB decklist links
- **NetrunnerDB** — ✅ reviews endpoint (community card analysis), MWL/ban list history, rulings
- **Null Signal Games** — prose only (ban reasoning, format definitions), would need scraping
- **YouTube** (Metropole Grid, Neon Static, Aksu, DullBulb) — title/description scraping for topic signals, transcripts for qualitative meta. Phase 3+ priority.

**Key new capabilities scoped:**
- Deck archetype classification (glacier, rush, asset spam, kill, etc.)
- Card synergy detection via PMI co-occurrence analysis
- Format legality (rotation, bans, restrictions)
- 7 new quiz types (archetype, synergy, deck context, legality, meta shift, counter)

**Important tension identified:** This scope is heavy infrastructure — the exact trap from Cycle 1. Resolution: two parallel tracks. Track A (sessions 1-3): agent behavior on existing v1 data. Track B (sessions 4+): meta enrichment per the scope doc. Don't let Track B steal from Track A.
