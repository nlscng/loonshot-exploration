## Scope Update: Adding "Meta" Knowledge

**New goal:** The agent shouldn't just quiz on card text — it should understand the **meta** (community evaluation of cards, popular decks, tournament performance).

### Meta Data Sources Researched

| Source | What It Provides | API Status |
|--------|-----------------|------------|
| **NetrunnerDB decklists API** | Published decklists with `tournament_badge`, card counts, player descriptions with results | ✅ Working — `/api/2.0/public/decklists/by_date/{date}` |
| **AlwaysBeRunning.net API** | Tournament listings (location, type, date, players) | ⚠️ Partial — `/api/tournaments` works, other endpoints flaky |
| **KnowTheMeta.com** | Was a Netrunner meta site | ❌ Dead — now a Swedish gardening blog |

### How Meta Integrates

- **Card popularity:** Which cards appear most frequently in tournament-winning decklists? ("This card is in 80% of top Corp decks")
- **ID/archetype trends:** Which identities are winning? What archetypes dominate?
- **Card evaluation context:** Agent can explain *why* a card matters competitively, not just what it does
- **Quiz enrichment:** "This card is a staple in competitive Weyland decks — what does it do?" or "Name the most-played Jinteki ice in recent tournaments"

## Revised Component Plan

### Component 1: Card Data Layer *(unchanged)*
**What:** Fetch and cache Netrunner standard card pool from NetrunnerDB v2 API into local SQLite.
**API:** `netrunnerdb.com/api/2.0/public/cards` (v2, since v3 preview returned 500)
**Done when:** Can query cards by faction, type, side, and get full card text locally.

### Component 2: Agent Skeleton *(unchanged)*
**What:** Minimal working agent using MS Agents SDK (Python) that handles conversation turns via CLI.
**Done when:** Can run the agent, type a message, get a response.

### Component 7: Meta Data Layer *(NEW)*
**What:** Fetch and cache tournament decklists from NetrunnerDB + tournament info from AlwaysBeRunning.net. Compute card popularity stats (frequency in tournament decks, win rates where available).
**API sources:**
- NetrunnerDB: `/api/2.0/public/decklists/by_date/{date}` — iterate over recent dates, filter `tournament_badge: true`
- AlwaysBeRunning.net: `/api/tournaments` — tournament metadata
**Parallel:** Yes — independent of Components 1 and 2.
**Done when:** Can query "top 20 most-played Runner cards in tournament decks from the last 3 months."

### Component 3: Quiz Tool *(updated)*
**What:** Agent tool that generates quiz questions from card pool AND meta knowledge.
**Meta-enhanced questions:**
- "This card appears in 75% of competitive Criminal decks. What is it?" (meta → card)
- "Is [card] currently popular in tournament play?" (meta awareness)
- "Which faction's ICE is most dominant in recent tournaments?" (meta analysis)
**Depends on:** Components 1 + 2 + 7.
